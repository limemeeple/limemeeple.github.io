    프로그래밍 언어

    💻 프로그래밍 언어와 아키텍처의 이해

    개발자의 기본 소양 다지기

    1. 프로그래밍 언어란?

    프로그래밍 언어(Programming Language)는 인간이 컴퓨터에게 특정 작업을 수행하도록 명령을 내리기 위해 사용하는 인간과 컴퓨터 사이의 의사소통 수단입니다.

    컴퓨터는 본질적으로 0과 1(기계어)밖에 이해하지 못하지만, 사람은 0과 1로 복잡한 논리를 작성하기 어렵습니다. 따라서 프로그래밍 언어는 사람이 이해하기 쉬운 문법으로 코드를 작성하면, 이를 컴퓨터가 이해할 수 있는 신호로 번역(컴파일 또는 인터프리트)해 주는 역할을 합니다.

    2. 프로그래밍 언어의 종류

    용도와 구동 환경에 따라 다양한 언어가 존재하며, 대표적으로 다음과 같이 분류할 수 있습니다.

    • JAVA: 기업형 애플리해케이션 및 웹 백엔드 개발에 가장 널리 쓰이는 객체지향 언어입니다. JVM 덕분에 플랫폼 독립적입니다.
    • JavaScript: 웹 브라우저를 움직이게 하는 프론트엔드 필수 언어이며, Node.js의 등장으로 서버 개발까지 영역을 확장했습니다.
    • Python: 문법이 간결하여 학습 능률이 높고, 데이터 분석, 인공지능(AI), 머신러닝 분야에서 독점적인 위치를 차지하고 있습니다.
    • C / C++: 하드웨어를 직접 제어할 수 있을 만큼 강력하며, 운영체제(OS), 임베디드, 고성능 게임 엔진 개발에 쓰입니다.

    3. 저급 언어(Low-Level) vs 고급 언어(High-Level) 비교

    구분 저급 언어 (Low-Level Language) 고급 언어 (High-Level Language)
    대표 언어 기계어, 어셈블리어 JAVA, Python, C, C++, JavaScript 등
    인간의 이해도 매우 어려움 (이진수 및 하드웨어 명령어 위주) 매우 쉬움 (인간의 언어 문법과 유사)
    컴퓨터의 이해도 번역 없이 바로 이해하거나 번역이 매우 빠름 컴파일러/인터프리터 등의 번역 과정 필수
    실행 속도 하드웨어 최적화가 가능하여 압도적으로 빠름 번역 오버헤드 및 가상머신 구동으로 비교적 느림
    생산성 매우 낮음 (코드 작성이 복잡하고 유지보수가 난해) 매우 높음 (가독성이 좋고 풍부한 라이브러리 지원)

    4. 클라이언트(Client)와 서버(Server) 구조

    우리가 인터넷을 이용할 때 일어나는 모든 상호작용은 클라이언트와 서버의 요청 및 응답 모델로 이루어집니다.

    📱 / 💻
    클라이언트 (Client)

    웹 브라우저, 앱
    (요청을 보내는 곳)

    요청 (Request) ➡️
    ⬅️ 응답 (Response)
    🖥️ / 🗄️
    서버 (Server)

    데이터베이스, API 서버
    (요청을 처리하는 곳)

    [웹 서비스 아키텍처: 클라이언트와 서버의 데이터 교환 흐름도]
    • 클라이언트(Client): 서비스를 이용하는 고객을 뜻합니다. 크롬 브라우저, 스마트폰 앱 등이 여기에 해당하며, 서버에 필요한 데이터(예: 로그인 요청, 게시글 조회)를 요청(Request)합니다.
    • 서버(Server): 서비스를 제공(Serve)하는 컴퓨터입니다. 클라이언트의 요청을 받아 유효한지 검증하고, 데이터베이스에서 필요한 데이터를 찾아 가공한 뒤 다시 클라이언트에게 응답(Response)해 주는 역할을 수행합니다.
